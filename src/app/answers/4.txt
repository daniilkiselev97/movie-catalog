- Ввод URL и анализ браузером
https://example.com/path
Браузер анализирует структуру URL:
Протокол (https://) → какой протокол использовать.
Домен (example.com) → куда отправлять запрос.
Путь (/path) → какой ресурс на сервере запрашивать.
Параметры и якоря (?q=123#section) → для передачи данных

- Проверка кэша
HTTP Cache (Браузерный кэш):
1.Браузер запрашивает ресурс с сервера.
2.Сервер отправляет заголовки кэширования:
Cache-Control
Expires
3.Браузер сохраняет ресурс в локальном кэше и использует его при повторных запросах, если он ещё действителен.

- Разрешение домена (DNS Lookup)
1.Браузер проверяет DNS-кэш, если IP уже известен.
2.Если IP неизвестен → отправляет запрос к DNS-серверу:
example.com → возвращает IP-адрес сервера.
3.Используются DNS-over-HTTPS (DoH) для безопасности и ускорения некоторых запросов.

- Установление соединения с сервером
1.TCP соединение:
- Через порт (обычно 80 для HTTP и 443 для HTTPS)

- TLS/SSL handshake (для HTTPS):
1.Обеспечивает шифрование соединения.
2.Проверяет сертификат сервера → защищает от MITM-атак.

-Отправка HTTP запроса 
GET /path HTTP/1.1
Host: example.com
User-Agent: ...
Cookie: ...
Accept: ...

Ускорение:
HTTP/2 Server Push — сервер сразу может отправлять связанные ресурсы (CSS, JS).
Кэширование — Cache-Control, ETag позволяют браузеру не загружать повторно статические файлы.

- Получение ответа от сервера
Сервер обрабатывает запрос и возвращает HTTP-ответ:
Код состояния (200 OK, 301 Redirect, 404 Not Found и др.)
Заголовки
Тело ответа (HTML, JSON, изображение и т.д.)

- Обработка HTML в браузере
1.Парсинг HTML:
- Создаётся DOM (Document Object Model).
- Каждое <div>, <img> и <script> → узлы в DOM-дереве.
2. Парсинг CSS:
- CSSOM (CSS Object Model) строится из <style> и внешних файлов.
- Создаётся Render Tree (комбинация DOM + CSSOM).
3. Рендеринг:
- Браузер вычисляет layout (позиции элементов) и paint (отрисовку на экране).
Ускорение:
-Critical CSS → стили для видимой части страницы загружаются первыми.
-Lazy-loading → изображения загружаются по мере прокрутки.
-Preload / Prefetch → заранее подгружаются важные ресурсы.

-Обработка JavaScript 
1.Браузер обнаруживает <script> и выполняет код.
2.Используются async/defer, чтобы скрипты не блокировали рендеринг.
Ускорение:
1.Tree shaking уменьшают размер JS.

- Основные проблемы:
1.CORS (Cross-Origin Resource Sharing)
JS из одного домена не может обращаться к другому без разрешения.
Заголовки Access-Control-Allow-Origin решают эту проблему.
2.XSS (Cross-Site Scripting)
-Злоумышленник вставляет вредоносный JS на страницу.
-Решение: Content Security Policy (CSP), экранирование данных.

HTTPS / TLS
- Без шифрования данные могут перехватываться.
